2. **All Create 3 base topics silent** (battery/odom/dock) — base was pointed at a stale discovery server `192.168.50.31` from the old network. **Fix:** repoint it to the Pi's wired IP `192.168.186.3:11811` via the base web config + restart app.

3. **Laptop discovery found topics but no data — attempt 1** — laptop super-client configured via XML profile silently didn't apply. **Fix:** use `ROS_SUPER_CLIENT=True` env var instead of XML.

4. **Discovery worked, still no data — real bug** — robot's FastDDS discovery-server database was corrupted; it received the laptop's endpoint announcements but didn't relay them. **Fix:** restart discovery service + robot ROS stack.

Underlying constraint behind all of it: multicast discovery can't cross the two different `/19` NUwave subnets, so it has to run unicast via the Discovery Server.
