# DepthAI / OAK-D-LITE XLink Errors & Boot Behavior on Raspberry Pi 4 + ROS 2 Humble

**TL;DR.** On the Pi-4 the OAK-D-LITE stalls (the "X_LINK / sys_logger_queue" gotcha #8 in CLAUDE.md) almost always trace to one of three host-side causes: (a) the `usbfs` ring buffer is too small for the high-bandwidth depth stream → set `usbcore.usbfs_memory_mb=256` (or higher); (b) USB power/cable marginality on USB3 → you already force USB2 (`i_usb_speed:=HIGH`), which is correct; (c) the XLink **watchdog** (default **4 s** on USB) firing when the host stalls (GIL/CPU saturation on the Pi), which makes the device **reboot back to the USB bootloader** and detach/re-attach — exactly the "alive but 0 Hz" symptom. The errors `No data on logger queue!` / `Couldn't read data from stream: 'sys_logger_queue' (X_LINK_ERROR)` / `Communication exception` are all the **same fatal event** (XLink link dropped); recovery requires reconnecting to the device, not just waiting. In depthai-ros, enable `i_restart_on_diagnostics_error: true` to auto-reconnect. Capture detail with `DEPTHAI_LEVEL=debug` (or `trace`) and the XLink log level.
