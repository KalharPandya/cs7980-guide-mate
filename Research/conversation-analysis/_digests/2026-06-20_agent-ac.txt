All defaults below are confirmed against the **`humble` branch source** (`camera_param_handler.cpp`, `sensor_param_handler.cpp`, `camera.cpp`), not the v3/RVC4 docs — the v3 docs disagree on several defaults (e.g. v3 says `i_usb_speed` defaults to `SUPER`/`SUPER_PLUS` and `i_nn_type` to `none`; the humble source actually ships `i_usb_speed=SUPER` and `i_nn_type=spatial`). ## The one thing that matters most for auto-recovery

The driver's **only** built-in self-healing path is diagnostics-gated and is **OFF by default**:

- A `SystemLogger` (diagnostics) node publishes device health. This node only exists when **`camera.i_enable_diagnostics: true`** (default `true` in humble).
- `Camera::diagCB()` watches diagnostics for an **error** status on the camera node's `sys_logger`. On error, **if `camera.i_restart_on_diagnostics_error: true`**, it calls `Camera::restart()`.
- `restart()` does `stop(); start();` — it **tears down and re-initializes the device**, which is exactly the "don't sit dead after a disconnect" behavior you want. If it can't come back it logs `"Restarting camera failed."`
